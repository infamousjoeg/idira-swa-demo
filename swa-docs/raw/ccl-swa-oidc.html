Configure OIDC issuer values for SWA integrations

\n

This topic describes the OpenID Connect (OIDC) issuer and discovery values that Secure Workload Access (SWA) uses for external platform federation.

\n

Use this topic to review issuer and discovery values before you configure a supported external platform integration.

\n \n \n \n \n \n \n \n \n \n
 \n

For current regional availability and support scope, see Secrets Manager support and scope.

\n
\n

Use the required OIDC valuesCopy bookmark

\n

Use the following OIDC values when you configure a supported external platform integration.

\n
    \n
  • \n

    Issuer URL: Identifies the Secure Workload Access (SWA) trust domain for the integration.

    \n
    • \n
  • \n

    Discovery document: Available at <issuer-url>/.well-known/openid-configuration.

    \n
    • \n
\n

You can retrieve the discovery URL and JWKS URI from the trust domain object.

\n
To retrieve OIDC discovery values for a trust domain:
\n
    \n
  1. \n

    Authenticate to Secrets Manager - SaaS and get an access token.

    \n

    For details, see Authenticate user.

    \n
    2. \n
  2. \n

    Set environment variables for your tenant and trust domain.

    \n \n \n \n \n \n \n \n \n \n
     
    export TOKEN=\"<access-token>\"\nexport TENANT_SUBDOMAIN=\"<subdomain>\"\nexport TRUST_DOMAIN_NAME=\"<trust-domain-name>\"
    \n
    \n
    3. \n
  3. \n

    Send a GET request to the trust domain endpoint.

    \n \n \n \n \n \n \n \n \n \n
     
    curl -sS -X GET \"https://${TENANT_SUBDOMAIN}.secretsmgr.cyberark.cloud/api/swa/trust-domains/${TRUST_DOMAIN_NAME}\" \\\n  -H \"Authorization: Token token=\\\"${TOKEN}\\\"\" \\\n  -H \"Accept: application/x.secretsmgr.v2+json\" \\\n  -H \"Content-Type: application/json\"
    \n
    \n
    4. \n
  4. \n

    In the response, find the discovery URLs under jwt.discovery_endpoints.

    \n
      \n
    • \n

      OIDC discovery document: jwt.discovery_endpoints.oidc_discovery_url

      \n
      • \n
    • \n

      JWKS URI: jwt.discovery_endpoints.jwks_uri

      \n
      • \n
    \n
    5. \n
  5. \n

    Confirm your response includes oidc_discovery_url and jwks_uri values.

    \n \n \n \n \n \n \n \n \n \n
     
    {\n  \"name\": \"<trust-domain-name>\",\n  \"jwt\": {\n    \"discovery_endpoints\": {\n      \"jwks_uri\": \"https://<subdomain>.secretsmgr.cyberark.cloud/api/swa/trust-domains/<trust-domain-name>/.well-known/jwks\",\n      \"oidc_discovery_url\": \"https://<subdomain>.secretsmgr.cyberark.cloud/api/swa/trust-domains/<trust-domain-name>/.well-known/openid-configuration\"\n    },\n    \"signature_algorithm\": \"RS512\",\n    \"signing_key_type\": \"RSA_4096\"\n  }\n}
    \n
    \n

    This example shows the default RSA pairing (RS512 with RSA_4096). Elliptic-curve signing is optional on the trust domain. For allowed values, see Configure JWT requirements for SWA integrations.

    \n
    6. \n
\n

Use these OIDC values in the provider-specific integration steps for the supported external platform.
